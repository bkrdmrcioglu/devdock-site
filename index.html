FAQ

Short answers. No support ticket required for the obvious ones.

What is DevDock?

A native macOS app that scans your project folders, detects frameworks, and starts / stops local stacks (API, web, mobile) with logs, ports, and workspaces — without Docker Desktop theater.

Which macOS versions work?

macOS 14 Sonoma or later. Apple Silicon and Intel via the universal / arm64 Developer ID build we ship today.

Free vs Pro?

Free: up to 3 projects, Start / Stop / Open, scan, updates. Pro ($29 one-time): unlimited projects, workspaces, menu bar controls, external port awareness, auto workspace suggestions.

How do I install?

Homebrew: brew install --cask bkrdmrcioglu/devdock/devdock — or download the notarized zip from this site and drag DevDock to Applications.

Gatekeeper said it couldn’t verify the app?

Current releases are notarized Developer ID builds. Prefer Homebrew or the latest zip from this page. If you still see a warning on an old download, get 0.2.4+ again.

How do updates work?

DevDock checks public GitHub Releases. When a newer version is available, tap Update. It runs brew install --cask bkrdmrcioglu/devdock/devdock, quits, and opens the new app from Applications. You can still download the zip manually.

Does Start open Xcode / Android Studio?

No. For JS mobile (Expo / RN) it starts Metro and can send Simulator / Emulator keys. For pure Swift / Kotlin projects it builds and launches on Simulator / Emulator when tools are installed.

Why isn’t my Expo android folder a separate project?

On purpose. Nested android/ and ios/ under Expo / React Native belong to the JS app — listed once as Expo / RN so Start runs Metro, not a stray Gradle shell.

Will it pick my physical phone?

Quick targets (iOS Simulator / Android Emulator) never fall back to a physical device. Connected devices stay in their own list if you want an exact target.

Can it see projects I started in Terminal?

Yes. If the port is live and the listener cwd matches the project, DevDock shows it as external (yellow) so you don’t double-start.

Where is source code?

The Swift app source stays private. Only the notarized Mac binary (and this site) are public — same model as many indie Mac tools.

Payments / license?

Pro is sold via Lemon Squeezy. After purchase you get a license key to activate in Settings. One-time payment — no subscription.

Data / privacy?

Projects run on your machine. Scan roots and settings stay local. Update checks hit GitHub Releases; Pro activation talks to Lemon Squeezy. No project source is uploaded.
